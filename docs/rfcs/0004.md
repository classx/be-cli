## Problem

Основная ценность продукта — фокусное отображение текста. Без отдельной подсистемы viewport
логика центрирования и контекста расползётся между input/render и станет трудно поддерживаемой.

## Goal

Определить алгоритм расчёта видимого окна: активная строка в центре, контекст сверху/снизу.

## Design

### Входные данные

- Буфер и текущая позиция курсора (`line`).
- Размер терминала (`width`, `height`).
- Настройки: `lines_before`, `lines_after`.

### Выходные данные

- Диапазон видимых строк буфера.
- Индекс активной строки внутри viewport.
- Типы строк для рендера: `active`, `context`, `padding`.

### Правила MVP

- При достаточной высоте активная строка визуально центрируется.
- У краёв файла применяется clamp без выхода за границы.
- Если терминал слишком низкий, приоритет — активная строка + максимально возможный контекст.

### Поведение при resize

- На событие `Resize` viewport пересчитывается немедленно.
- Позиция курсора в буфере не меняется, меняется только видимое окно.

## Alternatives

1. **Показывать весь экран без фокусного окна**
   - Плюсы: проще реализация.
   - Минусы: не решает основную проблему RFC-0001.

2. **Фиксированная позиция курсора без центрирования**
   - Плюсы: меньше пересчётов.
   - Минусы: хуже UX для заметок.

## Voting

Предлагаемое решение: принять viewport с центрированием и настраиваемым контекстом.

Статус голосования: `pending`.

## Migration

Миграция не требуется.
