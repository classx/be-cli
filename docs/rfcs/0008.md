## Problem

Даже минималистичный редактор требует discoverability: пользователь должен быстро увидеть,
какие действия доступны и какие клавиши за что отвечают.

## Goal

Определить MVP-политику горячих клавиш и поведение экрана помощи.

## Design

### Набор hotkeys MVP

- `Ctrl+S` — сохранить
- `Ctrl+Q` — выйти
- `Ctrl+O` — открыть настройки
- `?` — открыть помощь

### Экран помощи

- Открывается по `?` как оверлей/отдельный режим.
- Показывает:
  - список горячих клавиш;
  - краткое описание действий;
  - пометку readonly-режима (если активен).
- Выход из help: `Esc` или повторное `?`.

### Политика конфликтов

- В MVP набор hotkeys фиксированный.
- Post-MVP допускается переназначение, но запрещены конфликтующие привязки.

## Alternatives

1. **Только README без встроенной помощи**
   - Плюсы: нулевая реализация в приложении.
   - Минусы: плохой UX в интерактивном режиме.

2. **Большой интерактивный help с секциями и поиском**
   - Плюсы: мощно.
   - Минусы: лишняя сложность для MVP.

## Voting

Предлагаемое решение: принять компактный встроенный help и фиксированные hotkeys для MVP.

Статус голосования: `pending`.

## Migration

Миграция не требуется.
