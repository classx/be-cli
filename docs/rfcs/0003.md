## Problem

Для CLI-редактора нужна предсказуемая обработка клавиш в разных Linux-терминалах.
Без формализации ввода невозможно надёжно поддерживать горячие клавиши и навигацию.

## Goal

Определить минимальную, устойчивую систему терминального ввода для MVP.

## Design

### Режим терминала

- Переключение терминала в raw mode на время работы редактора.
- Восстановление исходных настроек терминала при любом выходе (нормальном и аварийном).

### Набор клавиш MVP

- Редактирование: печатные символы, `Enter`, `Backspace`.
- Навигация: `ArrowUp`, `ArrowDown`, `ArrowLeft`, `ArrowRight`.
- Действия:
  - `Ctrl+S` — сохранить;
  - `Ctrl+Q` — выход;
  - `Ctrl+O` — настройки;
  - `?` — помощь.

### Контракт событий

Подсистема возвращает нормализованные события типа:
- `Insert(char)`
- `Action(Save|Quit|OpenSettings|Help)`
- `Move(Up|Down|Left|Right)`
- `Resize(width,height)`
- `Unknown(sequence)`

### Ошибки и отказоустойчивость

- Неизвестные последовательности не приводят к падению.
- При ошибке чтения из stdin показывается понятное сообщение и корректно восстанавливается терминал.

## Alternatives

1. **ncurses/полный TUI-фреймворк**
   - Плюсы: меньше low-level кода.
   - Минусы: дополнительная зависимость и менее прозрачный control flow.

2. **Собственный low-level парсер (выбранный путь)**
   - Плюсы: полный контроль и минимализм.
   - Минусы: больше ручной работы.

## Voting

Предлагаемое решение: принять нормализованный event-layer и ограниченный набор клавиш MVP.

Статус голосования: `pending`.

## Migration

Миграция не требуется.
