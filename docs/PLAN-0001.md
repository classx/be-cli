# PLAN-0001 — Реализация RFC-0001 на Hare

Связанный RFC: `docs/rfcs/0001.md` (status: `review`)

## 1) Цель

Собрать минималистичный Linux CLI-редактор для фокусного редактирования заметок:
- активная строка в центре;
- ограниченный контекст сверху/снизу;
- минимальный UI и быстрые клавиши.

## 2) Границы

### MVP
- Открытие файла: `be <path>`
- Базовое редактирование (вставка, удаление, Enter, Backspace, стрелки)
- Сохранение `Ctrl+S`, выход `Ctrl+Q` (с подтверждением при несохранённых изменениях)
- Статус-строка:
  - слева: `[filename] [saved|modified]`
  - справа: `Ln <line>, Col <col>`
- `Ctrl+O` — настройки
- `?` — помощь
- Настройки MVP:
  - `lines_before`
  - `lines_after`
  - `cursor_on_open: start | end`

### Post-MVP
- Тема
- Переопределение горячих клавиш
- Горячая клавиша «показать весь файл» (preview + скроллинг)

## 3) Технический подход (Hare)

- Ввод: raw mode (termios), чтение клавиш/escape-последовательностей
- Рендер: ANSI escape codes, перерисовка кадра
- Модули:
  - `input`
  - `buffer`
  - `viewport`
  - `renderer`
  - `fileio`
  - `settings`
  - `help`
- Linux-first (xterm/kitty/alacritty/gnome-terminal)

## 4) План по этапам

## Этап 0 — Каркас
- Структура проекта и точка входа
- Парсинг аргументов CLI (`<path>`, `--lines`, `--readonly`)
- Загрузка файла в буфер

**Готовность:** бинарник открывает файл и завершает работу без редактирования.

## Этап 1 — Ядро редактирования
- Буфер строк + курсор
- Операции редактирования
- Навигация стрелками

**Готовность:** можно редактировать текст в памяти, без финального UI.

## Этап 2 — Фокусный viewport
- Центрирование активной строки
- Контекст `before/after`
- Приглушение соседних строк

**Готовность:** на экране стабильно виден фокусный фрагмент.

## Этап 3 — Статус-строка и действия
- `[filename] [saved|modified]`
- `Ln/Col`
- `Ctrl+S`, `Ctrl+Q`, подтверждение выхода

**Готовность:** полный MVP-цикл open/edit/save/quit.

## Этап 4 — Панели `Ctrl+O` и `?`
- Экран настроек MVP
- Экран помощи
- Применение настроек в рантайме

**Готовность:** настройки меняются и сразу влияют на viewport.

## Этап 5 — Стабилизация
- Проверка в нескольких терминалах Linux
- Фикс мерцания/артефактов
- Ошибки доступа к файлам и readonly-поток

**Готовность:** готово к статусу RFC `accepted`/`implemented`.

## 5) Риски и триггеры пересмотра языка

Триггеры перехода с Hare на Rust:
- значительная сложность TUI/ввода;
- сложные edge-cases Unicode/ширины символов;
- отсутствие нужных библиотек/инструментов;
- падение скорости итераций.

## 6) Минимальная проверка качества

На каждом этапе:
- ручной smoke-test (open/edit/save/quit);
- проверка статус-строки и `Ln/Col`;
- проверка `cursor_on_open=start|end`;
- проверка recovery при ошибке сохранения.

## 7) Definition of Done (MVP)

- Все пункты MVP из RFC реализованы;
- Работа подтверждена минимум в 2 Linux-терминалах;
- Документация (README/CHANGELOG) обновлена при релизе;
- RFC-статус меняется через `rfc-cli`.
