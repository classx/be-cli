# PLAN-0001 — Реализация RFC-0001 на Rust

Связанный RFC: `docs/rfcs/0001.md` (status: `review`)

## 1) Цель

Реализовать тот же функциональный объём RFC-0001, но на Rust с акцентом на устойчивость и расширяемость.

## 2) Границы

### MVP
- `be <path>`
- Базовое редактирование
- `Ctrl+S` / `Ctrl+Q` (+ подтверждение при dirty-состоянии)
- Статус-строка:
  - `[filename] [saved|modified]`
  - `Ln <line>, Col <col>`
- `Ctrl+O` (настройки), `?` (помощь)
- `lines_before`, `lines_after`, `cursor_on_open`
- Конфигурация:
  - путь: `$XDG_CONFIG_HOME/be/config.toml` (fallback `~/.config/be/config.toml`)
  - если файл отсутствует, создаётся автоматически при первом запуске с дефолтными значениями.

### Post-MVP
- Темы
- Настройка хоткеев
- Полнофайловый preview со скроллингом

## 3) Технический стек (предложение)

Минимальный практичный набор:
- `crossterm` — ввод/события/терминальный режим
- `unicode-width` — корректная ширина символов в терминале
- ANSI-рендер вручную (или облегчённый слой поверх `crossterm`)

Примечание: `ratatui` можно подключить позже, если ручной рендер станет слишком дорогим в поддержке.

## 4) Архитектура

Модули:
- `input`
- `buffer`
- `viewport`
- `renderer`
- `file_io`
- `settings`
- `help`

Поток: `Input -> Buffer -> Viewport -> Renderer`.

## 5) План по этапам

## Этап 0 — Bootstrap
- Cargo-проект
- CLI-аргументы
- инициализация конфигурации (чтение `config.toml` или автосоздание дефолтного файла)
- загрузка файла

**Готовность:** файл открывается, контур приложения рабочий.

## Этап 1 — Buffer + Cursor
- модель текста
- операции вставки/удаления
- навигация

**Готовность:** редактирование в памяти без полного UI.

## Этап 2 — Viewport
- центрирование курсора
- контекст до/после
- вычисление видимых строк

**Готовность:** корректный фокусный вывод при перемещении.

## Этап 3 — Renderer + Status
- отрисовка активной/соседних строк
- статус-строка в формате RFC
- подавление мерцания (батч-обновление кадра)

**Готовность:** стабильный интерактивный экран.

## Этап 4 — Actions
- `Ctrl+S`
- `Ctrl+Q` (+confirm)
- `Ctrl+O`
- `?`

**Готовность:** полный пользовательский сценарий MVP.

## Этап 5 — Hardening
- обработка ошибок ввода/вывода
- readonly режим
- smoke-тесты в разных терминалах Linux

**Готовность:** качественная MVP-версия для ежедневного использования.

## 6) Риски

- Возможный рост сложности из-за overengineering.
- Лишние зависимости, если не держать MVP-границы.

## 7) Контрольный список качества

- Корректная работа `Ln/Col` на всех действиях
- Проверка dirty-state (`saved|modified`)
- Проверка `cursor_on_open=start|end`
- Проверка сохранения и ошибок прав доступа
- Проверка производительности на больших заметках
